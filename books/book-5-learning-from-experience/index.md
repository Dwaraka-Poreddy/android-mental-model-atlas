# Book 5 — Learning From Experience

This book teaches through real-world case studies, production code examples, and architecture journeys.

**Assumes:** Books 1-4 complete (experienced Android engineer)

## Contents

- [x] [Offline-First Order Tracking (case study)](./01-offline-first-feature.md) — A full walkthrough of building an offline-first feature with Room, Flow, and WorkManager
- [ ] Architecture Journey: From MVP to Clean Architecture — How a real app's architecture evolved over time and why each decision was made
- [x] [Migration from XML to Compose (case study)](./03-migration-xml-to-compose.md) — A real migration from View-based UI to Jetpack Compose in an existing app
- [ ] Production Code Walkthrough: NowInAndroid — Annotated reading of Google's reference app and the patterns it demonstrates
- [ ] Case Study: Large-Scale Refactor — How teams plan and execute large refactors without breaking production
- [ ] Building a Feature End-to-End — Implementing a complete feature from API contract to UI using all Atlas concepts

## Reading

Each case study is self-contained. Read in any order based on what interests you.

## Goal

By the end of this book, you will:
- Have seen how experienced teams make architectural decisions
- Understand real-world trade-offs and compromises
- Recognize patterns in production code
- Know how applications evolve and scale
- Be ready to navigate any production Android codebase
