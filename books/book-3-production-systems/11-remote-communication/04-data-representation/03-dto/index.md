# Concept: DTO (Data Transfer Object)

**Purpose:** Understand how to translate API responses into application models

This concept is split into two parts because the mental model and production code are both substantial and distinct.

## Included Files

- [x] [Mental Model](./01-mental-model.md) — What a DTO is and why it exists
- [x] [Production Code](./02-production-code.md) — How DTOs appear in real Android code

## Reading Order

1. First, read the mental model to understand the concept
2. Then, read the production code to see how it works in practice

## How This Fits

DTOs are the boundary between external APIs and your application's domain models. Understanding this pattern is critical for building maintainable production code.
