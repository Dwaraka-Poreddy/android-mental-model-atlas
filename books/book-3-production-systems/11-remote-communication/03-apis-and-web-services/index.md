# Section: APIs and Web Services

**Purpose:** Understand how APIs work

This section covers API design, REST principles, and how web services expose data.

## Concepts Included

- [x] [API](./01-api.md) — A defined interface for how two systems communicate
- [x] [Web Service](./02-web-service.md) — An API exposed over HTTP that external clients can call
- [x] [Endpoint](./03-endpoint.md) — A specific URL that accepts requests for a defined operation
- [x] [REST and RESTful APIs](./04-rest-and-restful-apis.md) — The architectural style that uses HTTP verbs and URLs to model resources
- [x] [Authentication vs Authorization](./05-authentication-vs-authorization.md) — Proving who you are vs proving what you're allowed to do
- [x] [Tokens](./06-tokens.md) — Short-lived credentials used to authenticate API requests without sending passwords
